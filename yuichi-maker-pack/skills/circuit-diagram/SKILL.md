# 回路図作成スキル

## できること・できないこと（正直な線引き）

- **できる**：Pythonライブラリ**schemdraw**（動作確認済み・v0.23）で回路図をコード生成し、SVG/PDF/PNGで出力する。抵抗・LED・IC・スイッチ・電源などの電子回路記号のほか、遮断器（Breaker）・ヒューズ・変圧器・モーター・リレー等の要素があり、簡略化した単線結線図・系統図も描ける。修正はコードの書き換えなので、「抵抗値を変えて」「この分岐を追加して」に素早く対応できる。
- **できない/不向き**：JIS C 0617の配電用図記号への完全準拠（schemdrawはIEEE/IEC系の記号体系）。正式提出用・官庁提出用の設備図面は、このスキルの図を下書き・検討用とし、最終版はCAD（JW-CAD等)やKiCadでの清書を案内する。図面がどの用途か（検討用メモか、提出物か）を最初に確認する。
- **KiCad支援**：ユーザーはKiCadでの正式な回路図作成も行う（例：リチウムイオン電池劣化測定器）。KiCadの操作手順・シンボルライブラリの選び方・ERCエラーの解決などの相談にはテキストで支援する。

## 設計モード（機能・部品の指定から回路を起こす）

「◯◯する回路を、この部品で」という依頼では、作図の前に設計工程を挟む。

1. **仕様の確定** — 機能（何を入れて何が出るか）、電源電圧、負荷（インピーダンス・電流）、使いたい/手持ちの部品、性能の要求水準（趣味の実用レベルか、特性を追い込むか）を確認する。手持ち部品の活用が趣味では優先事項なので「指定部品で成立するか」をまず検討し、不足部品は代替案を添えて提示する。
2. **回路方式の選定** — 定番トポロジから選び、選定理由を一言で示す（例：ヘッドホン駆動→エミッタフォロワ＝電流増幅・低出力インピーダンス）。奇抜な独自回路より、実績ある定番回路＋定数調整を基本とする。
3. **定数計算の根拠を残す** — 抵抗・コンデンサ値は計算過程を示す（バイアス電流、カップリングのカットオフ f=1/(2πRC)、電流制限、時定数など）。「なぜこの値か」が残っていれば、ユーザーが後で自分で調整できる。
4. **定格チェック** — 部品の耐圧・許容損失・最大電流を確認し、余裕率を明記する。指定部品が定格不足なら作図前に指摘する。
5. **設計内容の確認を取ってから作図へ** — 回路方式と主要定数に合意してから、以降の「作図の進め方」に進む。設計と作図を一往復で全部やろうとせず、設計→合意→作図の順にする。

## 作図の品質管理（実戦で確立した手順・必ず守る）

LiPoチェッカー図面の反復修正で確立したノウハウ。これを守らないと「ラベルの重なり」「枠からの見切れ」が高確率で再発する:

1. **ラベルは絶対座標で置く** — 素子への`loc=`/`ofst=`相対指定は素子の向きで挙動が変わり事故のもと。値ラベル・部品名は`elm.Label().at((x, y))`で空き地に明示配置する。素子自体には`.label()`を付けない。
2. **ICは汎用ボックスより定番記号** — `elm.Ic`の箱は狭所でピン名と他ラベルが衝突しやすく、正体も伝わりにくい。TL431ならツェナー+REF線のようなホビー誌の定番表現を優先し、ピン名(K/A/REF等)は記号の外に小さな独立Labelで添える。
3. **生成後は衝突検出スクリプトで機械検証してから納品する** — schemdrawの純SVGバックエンド(`schemdraw.use('svg')`)で出力し、SVG内の全テキストbboxと長い線分・viewBox枠を突き合わせて、(a)テキスト同士の重なり (b)テキストと配線の重なり (c)枠からのはみ出し、を判定する。判定の要点: 複数行テキストは基準点から**下に**伸びる/CJK文字幅は約1.05×フォントサイズ・ASCIIは約0.58×/枠判定は幅推定誤差として±8pxを許容。衝突0件になるまで修正してから出す。実装例は過去会話のcollide2.py(LiPoチェッカー)を参照。
4. **右端に余白を確保する** — 右端付近のラベルは見切れやすい。ラベルは原則素子の左側に置き、レールは最右要素より1.5〜2.0ユニット長く引く。
5. **自分のプレビュー確認に頼らない** — 環境によっては生成画像の目視確認ができないことがある。機械検証を通した上で、最終確認はユーザーのスマホ表示で取る(スマホは横幅が狭く、PDFの横長図面の見切れが起きやすい)。
6. **修正は座標の理屈で行う** — ユーザーから「重なっている」と指摘されたら、感覚で動かさず、衝突検出器にかけて座標で原因を特定してから直す。

## 作図の進め方

1. **回路の内容を確定させてから描く** — 部品リスト（型番まで）、接続関係、電源電圧を確認する。曖昧なまま描き始めない。会話の流れで既に設計済みの回路（maker-codingやcub-ev-conversionで詰めた構成）なら、その内容をそのまま図に落とす。
2. **ピン割り当てを創作しない** — ICやモジュール（ESP32、INA226、モータードライバ等）の実際のピン名・ピン番号が必要な図では、web検索でデータシートを確認するか、確信が持てない場合はピン名をユーザーに確認する。**間違ったピン番号入りの回路図は、無いより有害**。
3. **schemdrawで生成 → プレビュー → 修正のループ** — 生成したSVGを提示し、レイアウトの読みやすさ（交差の少なさ、信号の流れが左→右、電源が上・GNDが下）を整える。
4. **出力** — 最終版はSVGとPDFの両方を/mnt/user-data/outputsに出す。ユーザーが後で自分で修正できるよう、**生成に使ったPythonコードも必ず一緒に渡す**。
5. 配線表（どの線をどこへ）と回路図はセットで整合させる。maker-codingスキルの配線説明と食い違う図を出さない。

## 図の種類別の方針

### 電子工作の回路図（Arduino/ESP32等）
- マイコンは長方形ブロック（`elm.Ic`）で表し、使用ピンだけをピン名付きで出す。全ピンを描いて煩雑にしない。
- 電源系（電池、レギュレータ、デカップリングコンデンサ）と信号系を視覚的に分ける。
- プルアップ/プルダウン抵抗、電流制限抵抗には値を必ずラベルする。

### バイクEV化・車両の配線図
- 主回路（バッテリー→ヒューズ→コントローラー→モーター）を太線・上段に、制御系（キー線、スロットル、ブレーキ信号）を細線・下段に描き分ける。
- ヒューズ定格・電線サイズ（sqmm）を線の近くにラベルし、配線表と一致させる。
- コネクタ位置（XT90等)も図中に明示すると、実作業で迷わない。

### 単線結線図・設備系統図（簡略版）
- 受電点→変圧器→幹線→分岐の流れを上から下へ。遮断器の定格（AT/AF）、変圧器容量、ケーブルサイズをラベルする。
- JIS記号との差異がある要素は凡例を図中に付けて誤読を防ぐ。
- 正式図面が必要な場合はこの図を「検討用」と明記し、CAD清書を前提とする。

## schemdrawの実装メモ（生成時に参照）

- 環境にない場合は `pip install schemdraw --break-system-packages` でインストールできる（動作確認済み）。
- **日本語ラベルの文字化けは解決済み**。以下のレシピを使う（Noto Sans CJK JPは環境に標準搭載、2026-07動作確認済み）:
```python
import matplotlib.font_manager as fm
fm.fontManager.addfont('/usr/share/fonts/opentype/noto/NotoSansCJK-Regular.ttc')
import schemdraw
import schemdraw.elements as elm
with schemdraw.Drawing(file='circuit.svg', show=False) as d:
    d.config(fontsize=11, unit=2.2, font='Noto Sans CJK JP')
    ...
```
- プレビュー用PNG化: `pip install cairosvg` → `cairosvg.svg2png(url='circuit.svg', write_to='circuit.png', scale=2.0)`。生成のたびにPNGを目視確認してからユーザーに出す。
- 動作確認済みの要素: `elm.BjtNpn(circle=True)`（トランジスタ、.base/.collector/.emitterアンカー）、`elm.Capacitor(polar=True)`（有極性コンデンサ）、`elm.BatteryCell`（電池）、`elm.Dot(open=True)`（端子・ジャック）、`elm.Ground()`。ホビー誌スタイルの回路図（例：ヘッドホンアンプ）を再現できることを実証済み。
- 基本形:
```python
import schemdraw
import schemdraw.elements as elm
with schemdraw.Drawing(file='circuit.svg', show=False) as d:
    d += elm.SourceV().up().label('5V')
    d += elm.Resistor().right().label('220Ω')
    d += elm.LED().down()
    d += elm.Line().left()
```
- 日本語ラベルはフォントによって文字化けする場合があるため、生成後にSVGを確認し、化ける場合は英数字ラベル＋別途凡例（日本語）の構成に切り替える。
- 複雑な図は`d.push()/d.pop()`や座標指定(`at()`, `to()`)で分岐を制御する。ICは`elm.Ic(pins=[...])`でピン定義する。
- PDF出力は`file='circuit.pdf'`を指定するだけでよい。

## 製作目的の依頼では「図面セット」で出す

ユーザーの主目的は、部品を購入してユニバーサル基板にはんだ付けして完成させること。回路図単体で終わらせず、以下をセットで納品する:

1. **回路図**（schemdraw生成、SVG+PDF+生成コード）— 部品番号（R1, C1…）と値・型番をすべてラベルする。この番号が部品表・配置の共通キーになる。
2. **部品表（BOM）** — 部品番号・品名・値/型番・数量・パッケージ（リード品前提）・入手先の目安（秋月電子等）と概算単価。**代替可の部品は代替条件を書く**（例：指定トランジスタ→hFE同等のNPNで可、耐圧◯V以上）。買い物メモとしてそのまま使える粒度にする。最新の在庫・価格が必要ならWeb検索で確認する。
3. **製作上の注意** — 有極性部品（電解コン・ダイオード・トランジスタの足）の向き、はんだ付け順序の定石（背の低い部品から）、テスト手順（電源投入前の導通確認→無負荷で電圧確認→負荷接続）。
4. **ユニバーサル基板の配置図（求められたら）** — 方眼グリッド上の部品配置とジャンパ経路の図。matplotlibのグリッド描画で作れるが、回路図ほど枯れていないので、まず配置方針（電源ラインは端に、入力と出力を離す、GNDは一点に寄せる等）をテキスト+簡易図で示し、必要に応じて図を詰める。

回路規模が小さい場合（部品10点程度まで）は配置図を省略し、回路図+BOM+注意事項の3点セットを標準とする。

**マイコン（Arduino/ESP32/Raspberry Pi等）を含む案件では、maker-codingスキルと併用し、上記セットに加えて「ピンアサイン表+完全なスケッチ/コード」まで含めた一式で納品する。ピン番号は回路図・表・コードの3箇所で一致させる。**

## レビュー（既存回路図の読み解き）を頼まれたとき

写真や図を渡されたら、(1)電源とGNDの経路、(2)極性のある部品の向き、(3)保護素子（ヒューズ・ダイオード）の有無、(4)定格の妥当性、の順に確認する。危険につながる誤り（逆接続、ヒューズなしの主回路、定格不足）を最優先で指摘する。
