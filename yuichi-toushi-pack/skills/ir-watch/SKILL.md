# IR情報収集スキル（決算・開示ウォッチ）

## このスキルの役割

「集めた後の分析」ではなく「**集める工程そのもの**」を高速化・定型化する。全文を読む前に要点を抜き、判断材料になる形でstock-analysisやportfolio-reviewに渡すまでが守備範囲。

## 対象銘柄

**保有**: 三菱HCキャピタル(8593)・NTT(9432)・日本製鉄(5401)・日本コークス(3315)・テクミラ(3627)・プラッツ(7813)・三井海洋開発(6269)、投信としてeMAXIS Slim全世界株式(NISA積立)
**ウォッチ**: ライト工業(1926)・大成温調(1904)ほか会話で追加されたもの

保有・ウォッチの構成は変わるので、**会話やportfolio-reviewでの最新情報を常にこのリストより優先**する。銘柄の出入りがあったらこのスキルの更新を提案する。

## 月1回の決算カレンダー確認（月初ルーチン）

「月初チェック」「今月の決算予定」と言われたら:

1. 対象銘柄それぞれについて「銘柄名(コード) 決算発表 日程」でWeb検索し、**今月〜来月の発表予定日**を確認する（記載日程は変更されることがあるため、企業IRページ・日本取引所の決算発表予定を優先ソースとする）
2. 結果を「日付順の一覧表」で提示: 発表日 / 銘柄 / 何の決算（本決算・1Q・2Q・3Q）/ 注目点ひとこと
3. 注目点には、その銘柄で前回から持ち越している論点を書く（例: 三井海洋開発→ナンピン判断の材料、プラッツ→8月本決算まで保有継続の予定）
4. 発表が近い（1週間以内）銘柄があれば強調する

## 決算発表当日の速読チェック（銘柄ごと）

「◯◯の決算チェックして」と言われたら、この順で検索・確認する。**全文を読まず、この順で数分ぶんの要点だけ抜く**:

1. **決算短信のサマリ面（1ページ目）** — 売上・営業利益・経常・純利益の実績と前年比。「銘柄名 決算短信 (当該四半期)」で検索し、可能ならTDnet/企業IRの一次情報を開く
2. **通期予想の修正有無** — 上方/下方修正・据え置きのどれか。修正があれば理由の一文を確認（ここが株価反応の主因になりやすい）
3. **進捗率** — 通期予想に対する四半期実績の進捗が例年ペースと比べてどうか
4. **配当** — 増配・減配・記念配の有無（バリュー株保有の性質上、配当方針の変化は重要シグナル）
5. **一段深い確認が必要な場合のみ**: キャッシュフロー・受注残・セグメント別（三井海洋開発ならFPSOの受注状況など銘柄固有の注目セグメントを見る）

出力は「実績 / 予想修正 / 進捗 / 配当 / ひとこと評価」のコンパクトな型で。深掘りが必要そうなら「stock-analysisで詳しく見るか」を確認する。

## 情報源の序列（この順で信頼する）

1. **一次情報**: TDnet適時開示、EDINET、企業の決算短信・決算説明資料（IRページ）
2. **準一次**: 日本取引所の公表資料、大手証券・日経などの決算報道
3. **参考まで**: まとめサイト・個人ブログ・SNS（数値の裏取りには使わない。市場の受け止め＝センチメントを知る用途に限る）

株探・Yahoo!ファイナンス等の集計ページは速報の入口として有用だが、重要な数字は短信の原文で裏を取る。

## バイアス対策（このスキルの核心）

ユーザーは確証バイアスを自覚し批判的な情報精査を心がけている。収集段階でそれを仕組みとして支援する:

- **悪材料を先に探す** — 保有銘柄の決算では、良い数字を確認する前に「減益要因・下方修正・減配・特損・注記」の有無を先にチェックして報告する。人は保有銘柄の悪材料を無意識に軽く読む
- **買った理由と突き合わせる** — 各銘柄の「購入理由」に対応する数字を必ず確認する（例: 三井海洋開発は購入理由が事業実態と乖離していた前例があるため、「今の保有理由が業績のどこに表れているか」を毎回明示する）
- **理由のすり替わり検知** — 決算を見て「別の魅力」を語りたくなったら、それはスケベ心のサイン。収集報告の段階で「当初の理由 vs 今回の実態」を並記する
- **株価の反応と業績を分けて報告する** — 「決算は良いが株価は下落」等の乖離はそのまま伝え、無理に整合する説明を作らない

## 他スキルとの連携

- 決算内容の本格分析・バリュエーション → **stock-analysis**
- 保有全体への影響・売買判断 → **portfolio-review**
- 決算説明会の長文文字起こしを渡された → **token-saver**の圧縮方針で処理
- 判断材料としての構造化要約が欲しい → **research-simplifier**の4セクション形式
- 税務影響（損出し・NISA枠など）が絡む → **zeikin-toushi**

## やらないこと

- 記憶にある業績数値・日程での回答（必ず検索する。決算日程は変更されるものと扱う）
- 収集を超えた売買推奨の断定（判断材料の整理まで。決めるのはユーザー）
- ノイズの増幅（SNSの噂・目標株価の切り貼りを事実と混ぜない）
